Deutsch's Algorithm

Quantum algorithms can be developed on top of a framework based on combining quantum gates. We can summarise the whole process in the following steps:

  • The system starts with bits in a classical state.
  • From there the system is put into a superposition.
  • The state is tranformed using unitary operations.(i.e. quantum gates)
  • Measure the qubits, forcing them to collapse in a state.

Notice each algorithm is not deterministic, having just a big probability of giving a good result.


The problem

Suppose there is a function f that maps from {0, 1} to {0, 1}. We wish to know whether the function is one-to-one or not.(i.e. f(0) does not equal f(1)) Opposing to the classic context, one can not examine the function on different results. The main idea is to consider the gate U as a matrix function operator(to be visualised as a gate) applying the function f to the given inputs.

The problem could be solved by evaluating f(0) ⊕ f(1). The xor function is the direct equivalent of asking whether f is an injection. Even though the function is evaluated twice in a classical context, the quantum context allows us to evaluate it only once.


Breaking down the problem

At the moment we know we want to obtain a superstate that will be extracted after measurement to f(0) ⊕ f(1). We only need a superstate that will be transfered to this particular point. The input is 0 and 1 along with any combination.

Firstly, we have to keep in mind that we can process two states simultaniously. We need a matrix(gate) that gives us a state that has an equal probability to be measured as a 0 or 1. Fortunately the Hadamard gate gives us the good behaviour.

Looking over the effect of the Hammond gate over the 0 and 1 states, we can observe that both bits will be driven to balaced superstates. You can observe that the products differ only by a sign.

At this moment, we have all the information: the input, the desired output, some means of transformations to balanced superstates and, finally the application of function f.


Putting the bits togeather

We will examine the circuit proposed by David Deutsch by looking at each step on the "pipeline".

The pipelineing mechanism can be expressed strictly in terms of matrices. We have 2 inputs that will be composed using the tensor product. Afterward each pair of binary gates will be multiplied(tensor product) so that we keep applying each matrix operator to the combined state.(4-by-1 matrix) Finally, the upper result will be measured.
